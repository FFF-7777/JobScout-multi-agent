# README 截图资产

当前目录保存主 README 使用的真实产品截图。新增或替换图片时应保持以下拍摄规范和命名规则。

## 通用拍摄规范

- 浏览器窗口建议使用 1440 × 900 或 1600 × 1000，同一组截图保持一致。
- 浏览器缩放固定为 100%，不要使用系统放大镜或页面缩放。
- 使用浅色主题，保留页面背景和导航栏，避免只截局部白色卡片。
- 所有简历、公司、联系人、手机号、邮箱和聊天内容必须脱敏。
- 截图前关闭浏览器下载提示、密码提示、开发者工具和悬浮通知。
- 不要展示 `.env`、API Key、OCR 密钥或完整本地文件路径。
- 推荐 PNG；不要反复压缩，不要使用带聊天软件水印的图片。

## 01 首页总览

- 文件：`01-home.png`
- 页面：`http://127.0.0.1:5173/`
- 必须包含：首页主卡片、模型状态、四个智能体卡片。
- 状态建议：模型已配置，连接状态正常。

## 02 简历画像

- 文件：`02-resume-profile.png`
- 页面：`http://127.0.0.1:5173/resume`
- 必须包含：简历导入入口、候选人信息、技能和项目。
- 数据要求：使用测试简历或彻底脱敏的真实简历。

## 03 岗位导入

- 文件：`03-job-import.png`
- 页面：`http://127.0.0.1:5173/jobs`
- 必须包含：导入方式、图片数量提示、基础/深度模式、岗位列表、开始分析按钮。
- 状态建议：至少准备 3 个岗位，其中 1 个选择深度分析。
- 补充图：`03-job-list.png`，用于展示更完整的结构化岗位列表。

## 04 执行流程

- 文件：`04-execution-flow.png`
- 页面：`http://127.0.0.1:5173/run`
- 必须包含：整体进度、处理中岗位、预计剩余时间、阶段卡片、联网能力状态。
- 状态要求：必须在任务运行过程中截图，不能使用已完成的静态页面代替。

## 05 推荐结果

- 文件：`05-results.png`
- 页面：`http://127.0.0.1:5173/results`
- 必须包含：筛选器、公司与岗位、匹配分、等级、投递建议和报告按钮。
- 数据建议：使用分数和决策存在明显差异的 3 至 6 个岗位。
- 补充图：`05-result-detail.png`，用于展示单个岗位的匹配证据、优势和缺口。

## 06 报告导出

- 文件：`06-reports.png`
- 页面：`http://127.0.0.1:5173/reports`
- 必须包含：历史报告列表、基础/深度标签、报告正文、导出按钮。
- 最佳状态：同时出现一份已完成报告和一份生成中的占位卡片。
